NetFlow (RFC 3954) — Cisco-originated protocol collecting IP traffic info from networks. Routers generate NetFlow records exported as UDP/SCTP packets to a NetFlow Collector. SIEM ingests flow data alongside log data for anomaly detection, bandwidth analysis, and C2 beacon identification.

Signature-Based Detection
Compares events against a database of known attack signatures. Detects intrusion attempts based on predefined patterns in IDS, WAF, and SIEM rules. Fast and reliable for known threats. Cannot detect novel attacks.
AspectSignature-Based
ApproachPattern / Rule matching
Detection typeKnown attacks only
Analytics typeSecurity analytics rules
DriverHuman-driven (rule authoring)
False positive rateLow (when tuned)
Zero-day detectionNo
Example toolsSnort IDS, ModSecurity WAF, SIEM regex rules
Update requiredFrequent signature updates
Example: Snort IDS Rule
alert tcp any any -> $HTTP_SERVERS $HTTP_PORTS (msg:"SQL Injection Attempt"; content:"' OR '1'='1"; nocase; sid:1000001; rev:1;)
Anomaly-Based Detection (UEBA)
Establishes a behavioral baseline and detects deviations from it. User and Entity Behavior Analytics (UEBA) uses machine learning to identify suspicious behavior patterns. Detects unknown and novel attacks.
AspectAnomaly-Based (UEBA)
ApproachBehavioral baseline deviation
Detection typeKnown + Unknown attacks
Analytics typeUser & Entity Behavior Analytics
DriverMachine learning driven
False positive rateHigher (tuning required)
Zero-day detectionYes (behavioral deviation)
Example toolsSplunk UBA, IBM QRadar UEBA, Elastic ML
Update requiredModel retraining / feedback loop
Example: UEBA Anomaly Rule
WHEN: user_login_time NOT IN baseline_hours[user] AND login_source_ip NOT IN known_ips[user] AND file_access_count > 3x baseline[user] THEN: risk_score += 40 Alert: "Behavioral Anomaly Detected — UEBA" Requires: manual investigation
Combined Approach — How Real SIEMs Use Both
Layer 1: Signature Rules
Fire on known patterns. Fast. Low latency. Used for known CVEs, attack signatures, and compliance checks. Generates most L1 alerts.
Layer 2: Behavioral Analytics
Continuous baseline monitoring. Slow burn detection over hours/days. Catches insider threats, APTs, credential abuse that signatures miss.
Layer 3: Risk Scoring
Combines signal from both layers. A user triggering 3 signature rules + UEBA deviation = high-risk incident. Reduces noise through correlation.
Log feeds alone are not enough. SIEM needs contextual data to improve situational awareness — enriching raw log events with user, asset, vulnerability, and threat information to determine if an event is truly malicious.
